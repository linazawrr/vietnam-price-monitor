# vietnam-price-monitor

Автоматический мониторинг цен на поездку Алматы → Нячанг (1–15 августа 2026, 6-9 дней, 2 взрослых).
Проверяет цены 2 раза в день (09:00 и 18:00 по Алматы) через GitHub Actions и присылает алерт в
Telegram, если находит что-то дешевле порога. Если ничего не найдено — молчит, лишних сообщений не будет.

## Что проверяется

- **Авиабилеты, регулярные** (`check_flights.py`) — через Travelpayouts/Aviasales API, маршрут ALA↔CXR.
  Порог: 320 000 тг/чел за туда-обратно. Цены в кэше API не всегда учитывают багаж — в алерте
  всегда есть напоминание об этом.
- **Авиабилеты, чартерные** (`check_charter_flights.py`) — ht.kz продаёт чартерные билеты отдельно
  от туров через `ht.kz/new/search/avia`, но эта страница рендерится только в мобильной версии и
  дёргает сторонний поисковый бэкенд (Bronix) с API-ключом, зашитым в JS-бандл ht.kz. Вместо того
  чтобы напрямую использовать чужой ключ в обход штатного сценария, скрипт водит настоящий headless-
  браузер (Playwright, мобильная эмуляция) по самой странице ht.kz и читает её собственные сетевые
  ответы — тот же доступ, что получает обычный посетитель с телефона. Проверяются только поездки на
  7 ночей (15 дат вылета) — полный перебор длительностей сделал бы прогон намного дольше. В ответе
  API уже есть точный вес багажа, поэтому напоминание "уточни багаж" тут не нужно — цена указана как есть.
- **Туры** (`check_tours.py`) — напрямую через JSON-эндпоинт `ht.kz/sellPage/loadTours`, который
  использует сам сайт ht.kz для страницы `/tours/vietnam-from-almaty` (найден через анализ сетевых
  запросов страницы). Region ID 63 = Нячанг, Country ID 7 = Вьетнам, Depart City ID 1 = Алматы.
  Порог: 800 000 тг за двоих (700 000 тг и ниже — отмечается как отличная цена). Отели фильтруются
  от 3 звёзд (`hotel.class` в ответе API).
  Когда находится подходящий тур, скрипт дополнительно запрашивает цену на того же самого отеля/дат
  для 1 взрослого через полноценный поисковый движок ht.kz (`ws.ht.kz/v1/search/web` — асинхронный:
  запрос кладётся в очередь, затем опрашивается до готовности) — это даёт реальную цену с учётом
  доплаты за одноместное размещение, а не просто половину цены за двоих. Обе цены попадают в один алерт.
  Запасной вариант, если эти эндпоинты когда-нибудь перестанут работать: Telegram-бот
  [@htsubscribtionBot](https://t.me/htsubscribtionBot) с горящими турами по Вьетнаму — проверять вручную.

## Настройка секретов (Settings → Secrets and variables → Actions)

| Secret | Обязателен | Как получить |
|---|---|---|
| `TELEGRAM_BOT_TOKEN` | да | уже настроен |
| `TELEGRAM_CHAT_ID` | да | уже настроен |
| `TRAVELPAYOUTS_TOKEN` | нет (без него проверка авиабилетов просто пропускается) | зарегистрируйся на [travelpayouts.com](https://www.travelpayouts.com), получи API-токен в личном кабинете, добавь как секрет |

## Запуск вручную

```
gh workflow run check_prices.yml
gh run watch
```

## Изменить пороги/даты

Все константы (даты, длительность, пороги цен) — в начале `check_flights.py` и `check_tours.py`.
