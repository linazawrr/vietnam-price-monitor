# vietnam-price-monitor

Автоматический мониторинг цен на поездку Алматы → Нячанг (2 взрослых для туров + 1 друг отдельно,
3 человека для билетов). Проверяет цены 2 раза в день (09:00 и 18:00 по Алматы) через GitHub Actions
и присылает алерт в Telegram, если находит что-то дешевле порога. Если ничего не найдено — молчит,
лишних сообщений не будет.

## Что проверяется

Для всех билетных источников (Aviasales, чартеры ht.kz, канал HT.KZ): вылет 1-20 августа, 3 человека,
длительность поездки 5-6 дней (короткая) или 8-9 дней (см. "мостик" ниже), порог — не дороже
1 000 000 тг на троих (округлено с 3 × 330 000), только прямые рейсы. Две необязательные пометки-бонусы:

- **🎯 Идеальные даты** — прилёт обратно в Алматы попадает на субботу/воскресенье.
- **🌉 Мостик через выходные** — и вылет, и прилёт попадают на пятницу-воскресенье. Смысл: если взять
  отпуск только на рабочую неделю (пн-пт, 5 дней), а вылетать/прилетать по краям ближайших выходных,
  фактическая поездка получается 8-9 дней при затрате всего 5 отпускных дней у друга.

- **Авиабилеты, регулярные** (`check_flights.py`) — через Travelpayouts/Aviasales API, маршрут ALA↔CXR,
  `direct=true` в запросе плюс проверка `transfers`/`return_transfers` на всякий случай. Цены в кэше
  API не всегда учитывают багаж — в алерте всегда есть напоминание об этом. Ссылка на бронирование
  сразу собирается на 3 пассажиров.
- **Авиабилеты, чартерные** (`check_charter_flights.py`) — ht.kz продаёт чартерные билеты отдельно
  от туров через `ht.kz/new/search/avia`, но эта страница рендерится только в мобильной версии и
  дёргает сторонний поисковый бэкенд (Bronix) с API-ключом, зашитым в JS-бандл ht.kz. Вместо того
  чтобы напрямую использовать чужой ключ в обход штатного сценария, скрипт водит настоящий headless-
  браузер (Playwright, мобильная эмуляция) по самой странице ht.kz и читает её собственные сетевые
  ответы — тот же доступ, что получает обычный посетитель с телефона. Прямые рейсы проверяются через
  поле `numberOfStopovers` в ответе (в среднем чартеры и так почти всегда прямые). Проверяются 4
  длительности (5,6,8,9 ночей) × 20 дат вылета = 80 запросов через браузер, поэтому этот шаг самый
  медленный (~9-10 минут). В ответе API уже есть точный вес багажа, поэтому напоминание "уточни багаж"
  тут не нужно — цена указана как есть.
- **Туры** (`check_tours.py`) — напрямую через JSON-эндпоинт `ht.kz/sellPage/loadTours`, который
  использует сам сайт ht.kz для страницы `/tours/vietnam-from-almaty` (найден через анализ сетевых
  запросов страницы). Region ID 63 = Нячанг, Country ID 7 = Вьетнам, Depart City ID 1 = Алматы.
  Даты вылета 1-15 августа, длительность 6-9 дней. Порог: 800 000 тг за двоих (700 000 тг и ниже —
  отмечается как отличная цена). Отели фильтруются от 3 звёзд (`hotel.class` в ответе API). Это
  отдельная логика от билетов — не расширялась вместе с ними, т.к. тур считается на пару (муж/жена)
  плюс отдельно на 1 взрослого (друг Рифат), а не на 3 человек сразу.
  Когда находится подходящий тур, скрипт дополнительно запрашивает цену на того же самого отеля/дат
  для 1 взрослого через полноценный поисковый движок ht.kz (`ws.ht.kz/v1/search/web` — асинхронный:
  запрос кладётся в очередь, затем опрашивается до готовности) — это даёт реальную цену с учётом
  доплаты за одноместное размещение, а не просто половину цены за двоих. Обе цены попадают в один алерт.
  Запасной вариант, если эти эндпоинты когда-нибудь перестанут работать: Telegram-бот
  [@htsubscribtionBot](https://t.me/htsubscribtionBot) с горящими турами по Вьетнаму — проверять вручную.
- **Авиабилеты, канал HT.KZ в Telegram** (`check_telegram_channel.py`) — канал
  [@HT_kz](https://t.me/HT_kz) регулярно постит подборки цен на билеты по разным направлениям
  (публично читается через `t.me/s/HT_kz`, без авторизации). Скрипт ищет в последних ~40 постах
  блок "🇻🇳 Вьетнам" ("Нячанг" и "Камрань" считаются одним и тем же направлением — это один и тот же
  аэропорт CXR, просто разные названия курортной зоны). Учитываются оба формата, которые встречаются
  в постах: уже готовый тариф туда-обратно одной строкой ("Алматы — Нячанг — Алматы") и раздельные
  билеты в одну сторону ("Алматы — Нячанг" / "Нячанг — Алматы"), которые скрипт сам объединяет в пару
  вылет+обратно (5-6 или 8-9 ночей). В алерте и дайджесте явно помечено, какой это тип: готовый тариф
  или самостоятельно собранная комбинация из двух билетов. У постов канала нет данных о пересадках,
  поэтому фильтр "только прямые" тут не применяется (в отличие от двух других билетных источников).

## Команды в Telegram по запросу

Кроме автоматических алертов, можно запросить дайджест топ-5 текущих вариантов в любой момент,
написав боту одну из команд:

- `/tours` — топ-5 туров ht.kz (цена за двоих и отдельно за одного)
- `/flights` — топ-5 авиабилетов Aviasales
- `/charter` — топ-5 чартерных билетов ht.kz
- `/tg` — топ-5 комбинаций билетов из Telegram-канала HT.KZ

Команды обрабатывает `handle_commands.py` в отдельном workflow (`handle_commands.yml`), который
опрашивает Telegram `getUpdates` каждые 15 минут — ответ приходит не мгновенно, а в течение
примерно 15 минут. Бот отвечает только на сообщения из чата с `TELEGRAM_CHAT_ID` — сообщения из
любых других чатов игнорируются. Чтобы 15-минутный опрос укладывался в бесплатный лимит минут
GitHub Actions, репозиторий сделан публичным (публичные репозитории получают неограниченные минуты
Actions) — токены и `chat_id` при этом остаются только в GitHub Secrets, в коде их нет.

## Настройка секретов (Settings → Secrets and variables → Actions)

| Secret | Обязателен | Как получить |
|---|---|---|
| `TELEGRAM_BOT_TOKEN` | да | уже настроен |
| `TELEGRAM_CHAT_ID` | да | уже настроен |
| `TRAVELPAYOUTS_TOKEN` | нет (без него проверка авиабилетов просто пропускается) | зарегистрируйся на [travelpayouts.com](https://www.travelpayouts.com), получи API-токен в личном кабинете, добавь как секрет |

## Запуск вручную

```
gh workflow run check_prices.yml
gh run watch
```

## Изменить пороги/даты

Все константы (даты, длительность, число пассажиров, пороги цен) — в начале каждого `check_*.py`.
